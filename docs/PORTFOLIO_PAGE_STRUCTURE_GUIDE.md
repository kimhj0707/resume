# 김현중 포트폴리오 페이지 구조/방식 가이드 (GitHub Pages / Static HTML 기준)

## 0) 전역 원칙 (모든 페이지 공통)

### 목적

- “개발을 이해하는 Technical PM”을 **3초 안에 이해**시키고,
- “그래서 이 사람을 뽑으면 뭐가 좋아지지?”에 **즉답**하게 만든다.

### 금지어/금지패턴

- 금지어: “아이디어”, “기획합니다”, “노력”, “열정”, “관심”
- 금지패턴: 기술만 나열(React, FastAPI…만), 결과 없는 설명, 주어 없는 문장(“~했음”)

### 강제 문장 규칙

- 모든 프로젝트/경험은 **문제 → 내 역할 → 구조 설계 포인트 → 결과** 순서 고정
- 주어는 “저는/내가” 혹은 “김현중은”으로 명확히
- 결과는 가능하면 **수치/상태/산출물**로 표현 (MVP 작동, 배포, 수상, 제출 등)

### 카드(콘텐츠 블록) 공통 레이아웃

- 제목(한 줄) + 한 줄 요약
- 본문은 4줄 이내로 핵심만
- CTA 버튼은 1~2개만: `Demo`, `GitHub`, `Docs`

## 1) Home (index.html)

### 목표

- 방문자가 3초 안에 “이 사람 포지션/강점”을 이해
- Projects/Competitions/Profile로 자연스럽게 이동

### 섹션 구성 (권장)

1. Hero
   - H1: 포지션 선언문(문제 해결형)
   - 1줄 서브카피(구조 설계/MVP/실행 강조)
   - 버튼 2개: GitHub / Projects
2. Core Skills (태그 4~6개)
   - “역할 중심” 단어로만 구성
3. Featured Projects (2~3개)
   - 가장 강한 2개만 노출
4. Featured Competitions (1~2개)
   - 수상/임팩트가 있는 것 우선
5. Footer

### Home 카피 체크리스트

- [ ] H1이 “무슨 일을 하는지” 즉시 말하나?
- [ ] “나는 어떤 문제를 줄이는 사람인지”가 드러나나?
- [ ] 프로젝트는 서비스 소개보다 “내 역할”이 앞에 나오나?

---

## 2) Projects (projects.html)

### 목표

- “일 시키면 구조부터 잡는 사람”이라는 확신
- 프로젝트가 많아도 **읽는 속도가 빨라야** 한다.

### 섹션 구성

1. 페이지 헤더: All Projects + 한 줄 설명
2. Project Card 리스트 (5개 내외)
3. 각 카드에 CTA: GitHub(필수), Demo/Docs(있으면)

### Project Card 포맷(강제)

- 프로젝트명 (한 줄)
- 한 줄 요약: “무엇을 해결하는지”
- 본문 4블록:
  1. 문제 정의
  2. 내 역할
  3. 구조 설계 포인트
  4. 결과/임팩트
- 하단: Tech Stack(최대 6개) — 단, 기술 나열이 아니라 “구조를 뒷받침하는 정도”로만

### Projects 페이지 체크리스트

- [ ] 서비스 설명이 길지 않은가? (길면 탈락)
- [ ] “내 역할”이 첫 화면에서 보이는가?
- [ ] 결과가 “작동/배포/수상/제출” 같은 상태로 명시됐는가?

---

## 3) Competitions (competitions.html)

### 목표

- 공모전 = 스펙이 아니라 “문제 정의 능력/기획 역량” 증명으로 보이게

### 섹션 구성

1. 페이지 헤더: Competitions & Awards
2. Competition Card 리스트
3. 카드마다 “의미 한 줄(Why it matters)” 강제

### Competition Card 포맷(강제)

- 공모전명 / 결과(수상·제출·본선 등)
- 한 줄 설명: 주제/문제 영역
- “의미 한 줄”: 내가 증명한 역량 (문제정의/구조화/검증/협업/발표 등)
- (선택) 산출물 링크: 발표자료/요약문서/레포

### Competitions 페이지 체크리스트

- [ ] 수상만 강조하지 말고 “내 기획 방식”이 보이는가?
- [ ] 공모전마다 ‘내가 뭘 증명했는지’ 한 줄이 있는가?

---

## 4) Profile (profile.html)

### 목표

- 경력 공백을 “일관된 사고 방식 + 실행 이력”으로 대체
- “함께 일하면 편하겠다(명확하다)” 느낌

### 섹션 구성

1. About Me (짧게)
2. Experience & Activities (3~5개 카드)
3. Working Style / Strength (선택)
4. Contact

### Experience Card 포맷(강제)

- 역할/타이틀 + 기간
- 핵심 불릿 3~5개
- 마지막 줄은 “결과/산출물”로 마무리

### Profile 페이지 체크리스트

- [ ] 에세이 톤이 아닌가? (선언형 문장)
- [ ] 강점이 “성격”이 아니라 “업무 결과”로 연결되는가?

---

## 5) 디자인/UX 최소 수정 가이드 (가벼운 톤으로 만들기)

- 카드 배경: 너무 무채색이면 CTA(버튼)만 튀어 보임 → 상단 Hero에 얇은 포인트 컬러 라인/그라데이션 추천
- 타이포: 제목 대비 더 키우고 본문은 줄이기(읽는 속도 ↑)
- 버튼: GitHub만 반복되면 지루함 → 가능하면 Demo/Docs도 1개씩 추가

```txt

PORTFOLIO_PAGE_STRUCTURE_GUIDE.md 규칙을 절대 어기지 마라.
각 페이지별로 섹션 구조를 유지하고, 카드 포맷을 강제 적용해라.
추상 문장 삭제, 문제→역할→설계→결과 순서 고정.
```
