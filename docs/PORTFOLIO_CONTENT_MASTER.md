### (1) K-푸드 수출 패스포트 AI

**한 줄 요약:** 수출 식품 라벨 규정 확인을 자동화하는 이미지 기반 분석 MVP

- **문제:** 수출 라벨 규정 확인이 국가별로 분절되어 있고, 라벨 텍스트 추출부터 검토까지 수작업 의존도가 높음
- **내 역할:** OCR→규정체크→홍보문구 생성 파이프라인 설계, MVP 범위 정의, 데모 시나리오 구성
- **구조 설계 포인트:** OCR 결과를 규칙(JSON)과 분리해 국가 확장 가능 구조로 설계, 입력(이미지)–처리–출력(리포트/문구) 흐름을 단순화
- **결과:** 이미지 업로드 기반 MVP 구현 단계, 발표/공모전 제출 가능한 형태로 산출물 정리

Tech(표기용, 최대 6): OCR / Rules(JSON) / LLM / Cloud Run(또는 배포) / Streamlit(or Web) / GitHub

### (2) 일드림 (Il-Dream)

**한 줄 요약:** 시니어 일자리 매칭의 ‘정보 시차·직무명 파편화’ 문제를 구조로 푼 매칭 서비스

- **문제:** “투자/필요 인력 발생”과 “채용공고 게시” 사이에 시차가 있고, 직무명이 제각각이라 매칭이 늦어짐
- **내 역할:** 문제 원인 3요소 구조화(직무명/시차/정보파편화), MVP 기능 정의, 매칭 흐름 설계
- **구조 설계 포인트:** 사용자 여정에서 “탐색→신뢰→연락” 단계의 병목을 정의하고, 추천/알림/리드 관리로 이어지는 흐름 설계
- **결과:** Tech for All 공모전 최우수상 수상 경험으로 기획 역량 검증, 포트폴리오 대표 프로젝트로 정리

Tech(표기용): User Flow / Matching Logic / MVP Scope / Docs / (필요시) React / DB

### (3) RealWalk

**한 줄 요약:** 상권/입지 의사결정을 돕는 GIS 기반 보행·POI 분석 서비스 기획

- **문제:** 입지 선정이 경험/감에 의존하며, “실제 유동/동선” 데이터를 구조적으로 비교하기 어려움
- **내 역할:** 데이터 요구사항 정의(보행/POI/건물입구), 분석 시나리오 설계, 서비스 화면 흐름 구성
- **구조 설계 포인트:** 지도 기반 탐색에서 “관측→비교→결정” 액션을 명확히 분리, 데이터 구조(PostGIS 등)와 화면 요구를 연결
- **결과:** 포트폴리오용 데모 시나리오 구성 및 기능 단위로 정리(탐색/필터/리포트)

Tech(표기용): GIS / POI / Data Modeling / (선택) PostGIS / API

### (4) WeatherFlick

**한 줄 요약:** 날씨 기반 여행지 추천을 마이크로서비스 관점으로 구조 설계한 여행 서비스

- **문제:** 여행 계획에서 날씨 변수가 가장 큰데, 정보가 분절되어 추천과 일정이 분리됨
- **내 역할:** 서비스 구조(프론트/백엔드/배치/관리자) 분리 관점 설계, 핵심 기능/플로우 문서화
- **구조 설계 포인트:** 추천(실시간)과 데이터 수집/가공(배치)을 분리해 확장 가능 구조로 설계
- **결과:** 최종 발표용 구조 문서/대본 정리, 포트폴리오용 아키텍처 사례 확보

Tech(표기용): Microservices / FastAPI / React / Batch / Admin

### (5) Venti Recipe (카페 레시피 PWA)

**한 줄 요약:** 매장 레시피 교육/운영을 위한 검색·옵션 기반 PWA 레시피 가이드

- **문제:** 레시피가 구두/문서로 흩어져 있어 신규 교육과 품질 유지에 비용이 큼
- **내 역할:** 레시피 데이터 구조(JSON) 설계, 옵션(달게 기본 포함 등) 규칙화, UI 흐름(검색/상세/옵션) 정리
- **구조 설계 포인트:** 레시피를 데이터로 표준화하고, 옵션 변경을 “stepOrder 기반”으로 관리해 유지보수 가능하게 설계
- **결과:** 실제 운영을 고려한 PWA 형태로 정리, 기능 단위(검색/테마/보안)로 지속 개선

Tech(표기용): PWA / JSON Schema / Search UX / (선택) React
